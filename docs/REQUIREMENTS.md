## 2. 핵심 파이프라인

```
YouTube URL 입력
  → 메타데이터 추출 (제목, 채널, 날짜, 길이)
  → 자막 추출 (ko > en, JSON3 포맷)
  → 영상 다운로드 (사용자 설정 화질, 기본 720p)
  → 프레임 캡쳐 (자막 구간별 / 장면 변화 / 고정 간격)
  → 슬라이드 생성 (slides.html)
  → Tauri 웹뷰에서 바로 열람
```

## 3. 캡쳐 모드 (3가지)

| 모드 | 설명 | ffmpeg 옵션 | 비고 |
|------|------|-------------|------|
| 자막 구간별 (**기본**) | 각 자막 세그먼트 시작 시점 캡쳐 | `-ss {ts} -frames:v 1` | 드롭다운에서 기본 선택 |
| 장면 변화 감지 | 화면 변화도 30% 이상 시 캡쳐 | `select='gt(scene,0.3)'` | 자막 없을 때 자동 폴백 |
| 고정 간격 | N초마다 캡쳐 | `fps=1/N` | 10/30/60초 프리셋 + 커스텀 값 입력 |

### 캡쳐 모드 선택 UI

- URL 입력 시 **자막 기반이 기본 선택**된 드롭다운 표시
- 사용자가 원하면 장면 변화 감지 또는 고정 간격으로 변경 가능

### 자막 없는 영상 처리

- 자막이 아예 없는 영상 → **자동으로 장면 변화 감지 모드(30% threshold)로 폴백**
- 사용자 개입 없이 자동 전환

### 자막 없는 프레임 표시

- 자막 없이 캡쳐된 프레임은 **타임스탬프만 표시** (예: `[00:01:30]`)

### 자막 언어 우선순위

- 한국어(ko) 우선 → 영어(en) 대체

## 5. 작업 큐 시스템

| 항목 | 결정 |
|------|------|
| 처리 방식 | **큐 방식 순차 처리** |
| 진행 상태 | **단계별 진행률 바 + 단계명** (예: "3/5 단계: 영상 다운로드 중... 45%") |
| 에러 처리 | **토스트 알림** 표시 후 다음 큐 항목으로 자동 진행 |

## 기술 스택

| 영역 | 선택 | 이유 |
|------|------|------|
| 프레임워크 | **Tauri** | Rust 백엔드 + 웹뷰 프론트. 번들 크기 작고 웹 기술 활용 가능 |
| 프론트엔드 | **바닐라 HTML/CSS/JS** | 프레임워크 없이 경량 구현 |
| 백엔드 로직 | **Rust** | Python 스크립트 전체 재작성. 자체 완결형 |
| 외부 도구 | **yt-dlp, ffmpeg 번들링** | 사용자 PC에 별도 설치 불필요 |
| 플랫폼 | **Windows 전용** | 빌드/테스트 단순화 |
| 배포 | **포터블 폴더** | 설치 프로그램 없이 폴더 복사로 동작 |

## 재작성 대상 (Python → Rust)

| Python 스크립트 | Rust 대체 | 핵심 로직 |
|----------------|-----------|-----------|
| `extract_metadata.sh` | yt-dlp CLI 호출 래퍼 | `--dump-json` 파싱 |
| `extract_transcript.sh` | yt-dlp CLI 호출 래퍼 | `--write-auto-sub --sub-lang ko,en --convert-subs json3` |
| `download_video.sh` | yt-dlp CLI 호출 래퍼 | `-f bestvideo[height<=720]+bestaudio` (화질 설정 반영) |
| `capture_frames.py` | ffmpeg CLI 호출 + JSON3 파싱 | 세그먼트 병합, `-ss` seek, `-frames:v 1` |
| `generate_output.py` | Rust 템플릿 엔진 | HTML 생성 (사이드바 목차 포함) |
