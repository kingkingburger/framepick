# framepick v0.1 Polish — Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

`to_asset_url()` 함수에서 상대경로일 때 `current_dir()` 대신 실행파일 디렉토리 기준으로 해결:

```rust
fn to_asset_url(path: &Path) -> String {
    let abs = if path.is_absolute() {
        path.to_path_buf()
    } else {
        // 실행파일 디렉토리 기준으로 상대경로 해결 (Tauri 앱에서 current_dir은 신뢰 불가)
        std::env::current_exe()
            .ok()
            .and_then(|p| p.parent().map(|d| d.to_path_buf()))
            .unwrap_or_else(|| std::env::current_dir().unwrap_or_default())
            .join(path)
    };
    // ... 나머지 동일
}
```

### Task 2: TOC 클릭 버그 수정

**원인 분석:**
- slides.html 내부의 TOC는 `<a href="#slide-{idx}">` 링크
- iframe srcdoc 안에서 해시 네비게이션이 부모 프레임에 전파될 수 있음
- `e.target.closest('.toc-link')` 는 정상이지만, `<a>` 태그의 기본 href 동작이 iframe 외부로 나갈 수 있음
- `e.preventDefault()` 가 호출되지만, srcdoc iframe에서 해시 링크 클릭 시 일부 브라우저에서 상위 문서 해시 변경

**Files:**
- Modify: `src/slides_generator.rs` — TOC 링크의 href를 `javascript:void(0)` 으로 변경하여 기본 네비게이션 방지
- slides.html 내부 JS의 TOC 클릭 핸들러는 이미 `e.preventDefault()` 하므로, href만 수정하면 충분

- [ ] **Step 1: slides_generator.rs — TOC 링크 href 수정**

`render_slides_html()` 함수 내 TOC 생성 부분에서 `href="#slide-{idx}"` → `href="javascript:void(0)"` 로 변경:

검색 패턴: `href=\"#slide-{idx}\"`
변경: `href=\"javascript:void(0)\"`

(data-slide 속성은 유지하여 JS 핸들러가 정상 동작)

- [ ] **Step 2: slides-viewer.js 샘플도 동일하게 수정**

`_generateSampleSlides()` 내 TOC 링크도 동일하게 수정.

- [ ] **Step 3: 기존 테스트 실행**

Run: `cargo test --lib`
Expected: 모든 테스트 PASS

- [ ] **Step 4: 커밋**

```bash
git add src/slides_generator.rs frontend/js/slides-viewer.js
```
